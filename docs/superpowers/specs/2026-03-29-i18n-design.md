# i18n Support: English + Spanish

## Overview

Add bilingual support (English/Spanish) to Andrea's portfolio using `@nuxtjs/i18n`. English is the default language at root URLs; Spanish lives under `/es/`. All user-facing content is translated except tech terms, company names, and product names.

## Decisions

- **Routing strategy**: `prefix_except_default` — EN at `/`, `/dev`; ES at `/es/`, `/es/dev`
- **Default language**: English. No browser detection, no auto-redirect. Manual switch only.
- **Switcher placement**: Inline at far-right of existing nav, separated by a thin `--divider` vertical line.
- **Translation scope**: All content translated except tech terms (see table below).

## Routing

| Page | English URL | Spanish URL |
|------|------------|-------------|
| Homepage (brand) | `/` | `/es/` |
| Developer page (recruiters) | `/dev` | `/es/dev` |

Existing English URLs remain unchanged. No redirects needed for current visitors.

## Locale Files

```
locales/
  en.json
  es.json
```

Lazy-loaded — only the active locale is bundled per page. Strings organized by section:

```json
{
  "nav": { "work": "...", "about": "...", "contact": "...", "recruiters": "..." },
  "hero": { "statement": "...", "label": "..." },
  "featuredWork": { "sectionLabel": "...", "projects": { ... } },
  "manifesto": { "statement": "...", "supporting": "..." },
  "about": { "sectionLabel": "...", "bio": "..." },
  "contact": { "heading": "...", "subheading": "...", "footer": "..." },
  "dev": { "heading": "...", "experience": { ... }, "projects": { ... }, "skills": { ... } },
  "seo": { "title": "...", "description": "...", "ogTitle": "...", "ogDescription": "..." }
}
```

## Language Switcher

**Location**: Inline at the far right of the nav in `Hero.vue`, after "For Recruiters", separated by a 1px vertical line using `--divider` color.

**Appearance**:
- Active language: `--text-primary` (#1A1A1A), `font-weight: 600`, `font-size: 11px`, `letter-spacing: 0.06em`
- Inactive language: `--text-muted` (#A39E99), same size
- Separator between EN/ES: not needed (spacing is enough) — but a thin divider separates the switcher from nav links

**Behavior**: Each label is a `<NuxtLink>` using `switchLocalePath()` from the i18n module. Clicking ES navigates to the Spanish version of the current page. Clicking EN returns to English.

**Mobile**: The switcher appears inside the mobile nav/menu if one exists. If the current mobile layout is just the desktop nav collapsing, the switcher follows the same pattern.

**On /dev page**: The dev page has its own nav. The switcher should appear there too, matching the same pattern.

## Editorial Translation Approach

The brand homepage copy (hero, manifesto, about) requires **adapted** Spanish — not literal translation. The goal is to preserve the editorial voice, warmth, and confidence.

Examples:
- EN: "I build digital spaces for people who care deeply about their craft"
- ES: "Creo espacios digitales para quienes cuidan cada detalle de su oficio"

- EN: "Your brand should feel like a natural extension of who you are"
- ES: "Tu marca debería sentirse como una extensión natural de quien sos"

Note: Use Argentine Spanish ("vos" form, "sos" instead of "eres") to match Andrea's identity and target audience.

## Technical Implementation

### 1. Install and configure `@nuxtjs/i18n`

Add to `nuxt.config.ts`:
- Module registration
- i18n config: `defaultLocale: 'en'`, `strategy: 'prefix_except_default'`
- Lazy loading with `langDir: 'locales'`
- Locale definitions: `{ code: 'en', file: 'en.json' }`, `{ code: 'es', file: 'es.json' }`

### 2. Create locale files

- `locales/en.json` — extract all current English strings
- `locales/es.json` — Spanish translations

### 3. Update section components

Replace hardcoded text with `$t()` calls in:
- `components/sections/Hero.vue`
- `components/sections/FeaturedWork.vue`
- `components/sections/Manifesto.vue`
- `components/sections/About.vue`
- `components/sections/Contact.vue`

### 4. Update /dev page

Replace hardcoded text in `pages/dev.vue` with `$t()` calls. Experience, projects, and skills data will reference locale keys.

### 5. Add language switcher

Add the EN/ES toggle to the nav in `Hero.vue` and the nav in `pages/dev.vue`.

### 6. Update SEO meta

Use `useI18n()` composable to get translated meta strings. Update `useSeoMeta()` calls in:
- `nuxt.config.ts` (global defaults)
- `pages/index.vue`
- `pages/dev.vue`

The i18n module automatically handles `<html lang>` attribute and hreflang alternate link tags.

## What Does NOT Change

- **Styling**: Zero CSS/Tailwind changes. Same palette, same typography, same spacing.
- **Animations**: Same scroll-reveal, same hero load animations.
- **Component structure**: No new section components. The switcher is a small addition to existing nav markup.
- **Layout**: Identical on both languages.
- **Deployment**: Same `yarn generate` + `firebase deploy`. The i18n module generates static pages for both locales.
- **Both domains**: `andrealopezpalomeque.dev` and `andrea-lopez-palomeque.web.app` continue working.
- **Assets**: No image or font changes.
